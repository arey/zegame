Create Tile Editor

This tool is a small editor that allow you to import your own bitmaps into ZeGame. In fact if you are an artist you can draw on Painter, Paint Shop, Photoshop or 3D Studio Max a picture and then use it to build map as ground tile or level decoration in the Map Editor. But before use them to design some levels, you have to define a set of properties for your bitmap as you could see in the following section.

In the File menu, you can load (Ouvrir decors) an existing .dec file to modify some of its parameters and of course then save (sauver decors) it. There is also a create (nouveau decors) menu you can use to import your own tile.
Here is the process to create your new tile :

  1. Import a bitmap

    The first thing you should do is to use the Chercher button to browse your drive in search of your .bmp file. Note that this tool just knows to load Microsoft bitmap file, so please convert your .gif, .jpg or .png file into a 256 colors .bmp.

    Once you have selected your tile file, you must enter in the Couleur transparente text field, the index of the background color that will not be displayed at screen and also its RVB composants. By convention I advise you to use the purple (255, 0, 255) color as transparent background color and placed it at the index number 0 of your color palette.
    The Selection button allow you to choose this color just by clicking on it in the right-top Decors panel.

  2. Choose an ID number

    The second step consist in selecting an ID number for this tile that has not already be taken by another tile. Then you have to not it in the IDTiles.id files that should list all the used ID of all the tiles of the game.


  3. Select tile ground zones

    If you already set down any tile on a game map inthe Level Editor, you should have noticed that the tile is delimited by one or more trapezoid shapes. They represents the projection of the object on the ground. So this is the part of the tile object that touch the ground level. It's like a 2D ground plan of an object modelized in 3D.

    If you are using a 3D software (like 3DS Max) to compute the tile, see the section that exeplains how to place your camera in your environment. And if you are using a 2D graphics software, you have to use the specific grid in refgrid2.gif to design the shape of your ground tiles.



reference grid scale (in pixels)

This wall is a representative tile sample to show you how to use the grid reference to setup then test if your own tiles have the right dimension.

So once you exactly placed the tile on the grid you have to select all the lozenges that belongs to it (and at least one). The first you place on the tile is the main diamond-shaped. MoO. It is a landmark for all the other lozenge you are going to place. Their coordinates are computed form this new center.

More the upper point of the main diamond-shaped is used to place its tile on the map during the game. You have to know that the sprites are drawn from the top of the screen to the bottom, along the Y axis. So before they are displayed the tiles are classified along a Y-Buffer. And the reference point of their main lozenge is used to sort them.

4. Set diamond-shaped information

Once you selected all the lozenges that belongs to the tile, you can set some information concerning them. First, you have to select one by using the "Infos Trapeze" button. The information about the one you selected will appear in this "Trapeze" panel :

Selected trapezoid information

  • num (fixed) : serial number of this diamond-shaped,
  • Ref U & Ref V (fixed) : its coordinates with regard to the main,
  • Affichage Partiel : allow you to divide your tiles in smaller vertical part,
  • Type de Terrain : ground type (wall, sand, mud, soil ...)
  • Type de Collisions : modify the behavior of collisions with characters,
  • Infranchissable : checked if we can walk on this trapezoid,
  • Cle d'affichage : indicate if this trapezoid is secondary for the display,
  • Hauteur : heigher point of this diamond-shaped (in pixels).

5. Why do you have to split big tile in vertical thin slices

When you have a big tile (that take more than 4 diamond-shaped), you should probably have to split it in several thin slices.You may ask why ? that's because of the 2D clipping. That I will try to explain in this section.

In some cases, a character or another sprite can be draw behind a big tile that didn't have been cuted even though it have to be over it in reality. Owing to the fact that the Y coordinate of this sprite is higher than the Y coordinate of the main trapezoid of the tile, so the sprite is displayed before the tile and the tile recovers it. The right side schema shows you that the monk is drawn before the wall because its Y coordinate is lower than the one of the wall.
