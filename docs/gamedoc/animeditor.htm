Anim Editor

This editor has been specially developped to edit the animations of characters and objects of this game. You can see in details each frame of the animation of a character and assign it a set of features used for collision detection or physics sinulation.
There is also a tool that allow you to import in ZeGame your own animations you have drawn on Painter, Paint Shop Pro, Photoshop, 3DS Max, Maya or Lightwave. It can be a simple fire or explosion animation as a 8 cardinal points animation of a dragon. First I recommend you to read the animation script file section that explains you how to write them.
The following documentation should help you to use this complex (and especially not very user-friendly) editor.

  1. Import an animation

    On 3D Studio Max 3.1, imagine that you have modelized a beautiful animation of a dragon that spit fire. To include it in the game, you have to render way to render it : either you render from just one point of view you want, either you render the dragon from 8 points of view, in fact from the 8 cardinal points : East, South-east, South, ....You will obtain 1 or 8 animation files (fli, flc, avi, mpeg , mov ...).

Next you have to save each indivual frame of your animation into bitmap files (.bmp). You can use the tool of your choice. I use a freeware software on Ms-Dos called Sea. Their name must have 8 characters and the name conventions are :

  • 1 or 2 letters for the point of view : s for south, ne for north-east, ...
  • 4 letters for the short name of your animation,
  • 2 or 3 letters for the index of the frame.

All those animation frames will be merge in big bitmap that I will call Animation Tile Bank. To do that, the AnimEditor has a small tool you can use. Put your files in the Temp\ sub-directory of the AnimEditor, launch it and click on the Anims >> Nouvelle Banque menu. The following dialog box should appear :


Animation Tile Bank DialogBox

All those fields are mandatory :

  • Lettres des bmps text field : 4 letters for the short name of your animation,
  • Designation text field : name of the output Animation Tile Bank bimap,
  • Nb sprites par animation text field : number of tile that composed an animation from one point of view,
  • Borne inferieure de l'intervalle text field : index of the first tile of your animation,
  • Format des bmp radio button : you can specify to the tools the name convention of your small tile bitmap. 8 characters is the one descibed above, 2 zeros if the index of a frame has 2 digits and 3 zeros if it has 3 digits,
  • Type de banque radio button : check 8 cardinal points if your animation has 8 point of views or 1 cardinal point if it just have one,
  • Space optimization check box : use a home-made algorithm to place the tiles on the bank,
  • Texture 256x256 check box : force the size of the bank to have 256x256 pixels large,
  • n coul fond text fields : 256 colors index (0 by default) and its RGB components of the transparent color of the tile,
  • n coul Ombre text fields : 256 colors index (255 by default) and its RGB components of the shadow color of the tile.
    •

    Click on the Ok button to go on the next step or Cancel to exit. You may get a tile bank like this below. This is the tile bank of a walking sword animation than we can use when the Hero walks and have its sword in hand. It was created with 16 sprites of an 8 point of views animation .


    Walking Sword bitmap animation tile bank


    Next you have to select for each sprite of your animation what I called the center of gravity of your object. It is used during transition between 2 animation to superimpose it. That way the connection between 2 images will be done easily without coding anything.
    For instance you can set this gravity point at the navel at a character.


    Anim Tile Bank Dialog

    When you will have done this last step, a .spi and a .bmp files will be created in the Temp\ sub-directory of this anim editor. Their name is the text you gave in the Designation text field.
    You could now use this animation in an existing character or sceneray object template, or create a new one as following...

    2. Create a new Template (for a Character or an object)

    This step allow you to create a model of character, a kind of class that your monsters and your NPCs could inherit. So they will have the same set of animations, but could not have the same behaviors.

    Go in the Files >>Nouveau >>Type de Personnage sub-menu.

    The First step involve to give an identifier string to this new class of character.

     

     

    Then you have to choose (or create if it soes not exist) a directory located in the Persos\ sub-directory of the ZeGame.

    All the files related to this model, will be place in this folder :

    • Main template file (.mdl)
    • Animation tile bank bimtap files (.bmp)
    • Animation tile bank information files (.spi)
    • Animation script file (.ina)

     

    You just now need to give a name for the main file of this object model which will have the .mdl extension. You can use the same name as the directory or the identifier.

    You have to create an empty animation script file (.ina). Go to Script >>Nouveau >>Script Animation menu. A header will appear in the left-bottom script text area. You can save it with the Script >>Save as menu. You must again choose a name for this file. It should be the same as previously.

     

    3. Open an existing Template

    To open a character model you have previously created, just go on Script >>Ouvrir >>Perso >>Chercher menu (Script >>Ouvrir >>Decors >>Chercher for other objects). A file dialog box will appear where you have to select in the good sub-directory the .ina file you want to open.
    For instance, if you want to open the Hero animation model, select the Persos\Hero\Hero.ina file.

     

    4. Add your animation to the active model

    In the previous step 1, you learned how to import your .flc, .mpg or .avi animation in the .bmp + .spi format used by ZeGame. At the end of the import those 2 files are located in the Temp\ sub-directory of the anim editor. To use this animation : copy the 2 files in the sub-directory of the character or object pattern of your choice.

    Whe you open an animation script file, its contents will appear in the left-bottom text area of the anim editor. You can either modify this script file in this area or in notepad, as you want ... For my part, I use notepad; but if somebody will implement .ina syntax highlighting, why not ?
